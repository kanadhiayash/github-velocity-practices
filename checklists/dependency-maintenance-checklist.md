# Dependency Maintenance Checklist

## Required

- [ ] Dependency alerts reviewed.
- [ ] High-severity issues triaged.
- [ ] Lockfiles committed.
- [ ] New dependencies justified.
- [ ] Unused dependencies removed.
- [ ] Major updates tested before merge.

## Recommended

- [ ] Dependency updates grouped by risk.
- [ ] GitHub Actions updates reviewed.
- [ ] Dependency-only pull requests separated from features.
- [ ] Temporary vulnerability exceptions have expiration dates.
