# Pull Request Readiness Checklist

## Required

- [ ] Pull request has one clear purpose.
- [ ] Summary explains what changed and why.
- [ ] Facts, assumptions, unknowns, and risks are separated.
- [ ] Required checks pass.
- [ ] No secrets or private data are present.
- [ ] Dependency changes are reviewed.
- [ ] Security-sensitive changes are labeled.
- [ ] Rollback note exists when behavior changes.
- [ ] AI-assisted changes are disclosed where relevant.

## Recommended

- [ ] Pull request size is reviewable.
- [ ] Screenshots or logs included where useful.
- [ ] Tests added or updated for behavior changes.
- [ ] Documentation updated in the same pull request.
